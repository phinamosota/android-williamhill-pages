# android-williamhill-pages

## 项目简介

本仓库用于归档和发布多个独立 HTML 页面。这些页面通常作为 Android 端功能模块的静态资源或测试页面使用，不针对任何单个域名或具体网站。仓库的主要目的是提供一个结构化的页面集合，方便版本管理和分发。

## 目录说明

```
android-williamhill-pages/
├── pages/          # 存放所有归档的 HTML 页面
├── assets/         # 页面依赖的静态资源（如 CSS、JS、图片等）
├── docs/           # 相关文档或说明文件
└── README.md       # 本文件
```

- `pages/`：每个 HTML 页面以独立文件或子目录形式存放，便于单独引用。
- `assets/`：公共资源文件，按类型或页面分组存放。
- `docs/`：包含页面使用说明、更新日志或其他辅助文档。

## 页面归档说明

- 所有页面均为独立 HTML 文件，可直接在浏览器中打开或嵌入 WebView 使用。
- 页面内容不包含特定域名或营销性质的链接，仅作为功能展示或测试用途。
- 归档的页面版本会随仓库更新记录，历史版本可通过 Git 提交历史回溯。

## 维护说明

- 新增页面时，请将 HTML 文件放入 `pages/` 目录，并在提交信息中简要说明页面用途。
- 若页面依赖外部资源，请一并放入 `assets/` 目录，避免使用外部 CDN 链接。
- 建议在 `docs/` 中维护一份页面清单，方便快速查找。
- 本仓库不承诺长期维护，但会尽量保持页面结构的稳定性。

## 如何使用

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/android-williamhill-pages.git
   ```
2. 直接打开 `pages/` 目录下的 HTML 文件查看效果。
3. 如需部署到服务器，可将整个仓库或 `pages/` 目录上传至静态托管服务。
